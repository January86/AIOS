# AIOS Codex Volume I — Foundation

## Declaration

AIOS is an Artificial Intelligence Operating System designed to become the operating layer for all software projects owned by the founder.

AIOS is not a chatbot.  
AIOS is not a single agent.  
AIOS is not only a dashboard.  
AIOS is an autonomous engineering organization expressed as software.

## Purpose

AIOS exists to help the founder:

- turn ideas into software,
- monitor existing projects,
- detect bugs and crashes,
- repair low-risk issues,
- coordinate AI agents,
- preserve knowledge,
- reduce manual operational burden,
- and evolve projects over time.

## North Star

> Build once. Evolve forever.

Every decision must answer:

> Does this make AIOS more autonomous, more reliable, or easier to evolve?

## Core Identity

AIOS has five identities:

1. Operating System
2. Engineering Organization
3. Software Factory
4. DevOps Operator
5. Knowledge Institution

## Non-Goals

AIOS will not initially attempt to:

- replace all human decision-making,
- give unlimited production autonomy,
- create hundreds of agents,
- prioritize visual novelty over backend truth,
- become dependent on one LLM provider,
- or implement future-vision features before the core is stable.

## Operating Principles

1. Foundation first.
2. Modular everything.
3. Event-driven coordination.
4. Mission-centric operation.
5. Model independence.
6. Policy-based autonomy.
7. Observable by design.
8. Memory before intelligence.
9. Human as founder.
10. Innovation backlog before scope creep.
