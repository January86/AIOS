# ADR-0051 — Lock AIOS Implementation Roadmap

## Context

AIOS planning has changed direction multiple times during early design. This helped exploration, but continuing to change the roadmap will create confusion and technical debt.

## Decision

The official roadmap is now locked in `ROADMAP-MASTER-001`.

Future roadmap changes require a new ADR.

## Rationale

AIOS must move from ideation to execution. Stable planning is required before implementation.

## Consequences

### Positive

- Clear execution path.
- Less confusion.
- Easier GitHub tracking.
- Easier handoff to AI agents and developers.

### Trade-off

- Some future ideas must wait.
- New ideas go to backlog, not active roadmap.
